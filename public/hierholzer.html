Hierholzer's Algorithm

The Hierholzer's Algorithm is a graph traversal algorithm that can find a Euler path or Hamiltonian cycle in a graph. It will attempt one traversal and will remove bridges when encountered onto a stack. If not all vertices are visited, it will "pop" what has been traversed, save it, and start a new path. Combine the paths from the last vertices to find a complete Euler path

  • Initialize two stacks, one "temp" path, one "final" path
  • Start at a vertices with 2 odd vertices, otherwise start at 0 odd vertices
  • Traverse until either all edges are visited (save to the final path and return the path)
  • Else, push what you have to the final path and start at any random vertices already encountered
  • Once you find a vertices that has a possible edge to traverse, begin
  • Traverse until no vertices are left (all have been popped onto the temp)
  • Assume graph is connected, repeat the previous path and connect to fine the Euler path

    • Complexity: O(n) -> Linear Time
